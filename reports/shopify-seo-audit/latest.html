Critical issues

These have the biggest effect on whether Google can find, understand, and rank the store correctly.

Critical

Store language is set to Swedish, but every page is written in English

All checkout/legal policy pages resolve with locale=sv and the site templates declare lang="sv", meaning Shopify's default storefront language is Swedish. But every product title, description, and SEO tag is in English, written for an English-speaking self-help audience, priced in USD. This sends Google a contradictory signal: the declared language says Swedish, the actual text says English. That mismatch can suppress visibility in English-language search results — the exact market this store is trying to reach — because search engines partly rely on declared language/locale to decide which market and language results to surface a page in.

Fix: In Shopify Admin → Settings → Languages, make English the store's primary/published language (or add English via Markets and set it as default for the audience you sell to). If Swedish is genuinely needed for a Swedish audience, set up proper multi-language / hreflang via Shopify Markets instead of leaving the mismatch as-is.
Critical

Every product image is missing alt text

Checked all 8 active products — every featured image and every additional product image has blank or null alt text. This is a 100% gap across the entire catalog. Alt text is one of the few remaining direct, easy levers for image search ranking and it also matters for accessibility (screen readers) and for context when images fail to load.

Fix: Add a short, descriptive, keyword-natural alt text to each product image, e.g. "The Self-Worth Reset — 7-day digital PDF workbook cover" instead of leaving it blank.
Critical

No category pages exist despite ready-made topic tags

The only collection in the store is the auto-generated "Home page" collection (handle: frontpage), and it has no SEO title or meta description set. Meanwhile every product is already tagged by topic — relationships, self-worth, money, parenting, health & wellness, career & business — but none of those tags have been turned into actual collection pages. That's 6 free, keyword-relevant landing pages the store isn't using, plus better internal linking and easier navigation for visitors.

Fix: Create a smart collection per tag (e.g. "Relationship Workbooks", "Self-Worth Workbooks"), each with its own SEO title/description targeting that topic, and link them in the main navigation.
Critical

Homepage has no meta description set

The shop-level SEO description field (used for the homepage's search snippet) is blank. Without it, Google will auto-generate a snippet from whatever text it finds on the page — usually worse at converting clicks than a written one.

Fix: Online Store → Preferences → set a homepage title and meta description (under ~155 characters) that states what the store sells and who it's for.

Important gaps

Important

Blog exists but has zero published posts

A "News" blog is set up but has no articles. This niche — relationship clarity, self-worth, money stress, parenting overwhelm — is unusually well suited to long-tail organic content (the kind of specific questions people type into Google before they're ready to buy). Right now the store has no content engine at all, and it's a brand-new domain with no existing authority, so organic growth will depend heavily on this.

Fix: Start a simple monthly content calendar — 2-4 posts/month answering real questions tied to each workbook topic, each linking to the relevant product/collection.
Important

Hard-coded, ever-expiring urgency copy

"The Clarity Starter Bundle" description states "$19.99 until Sunday midnight — then $29.99" as static text baked into the product page. If that deadline isn't actually enforced by an automation, it becomes a false claim every single week the page stays live — a trust and compliance risk, and a quality signal issue if visitors notice the "deadline" never actually arrives.

Fix: Either automate the real price change on that schedule, or remove the specific day/time and use evergreen framing (e.g. "limited-time launch price") that doesn't go stale.
Important

Repeated boilerplate opening line across most product pages

"Instant digital download. This product is delivered as a PDF file. No physical product will be shipped." appears verbatim as the opening line on 5 of the 8 product pages. Search engines handle small shared boilerplate fine, but it's a missed chance to make each page's opening lines more unique and topic-specific, which helps both SEO relevance and reader engagement.

Fix: Move that trust/format info into a shared visual badge or icon row (like the newer product pages already do), and let the opening paragraph be unique, benefit-led copy instead.

What's already working well

Good

Every product has a custom, well-sized SEO title and meta description

All 8 active products have unique seo.title and seo.description values, comfortably within Google's practical display limits (titles ~33–46 characters, descriptions ~78–101 characters). That's a solid foundation most new stores skip entirely.

Good

Clean, descriptive URL handles

Every product URL is short, hyphenated, and matches the product topic (e.g. /products/the-self-worth-reset) — no auto-generated numeric or junk handles.

Good

Consistent structure and semantic headings in product copy

Descriptions use H2/H3 sections ("What's inside", "Who this is for", FAQ blocks) rather than one wall of text — good for readability and a solid base for adding FAQ rich-result markup later.

Good

SSL and domain setup look correct

The primary domain (theclarityshop.se) has SSL enabled and is correctly set as the shop's primary domain in Shopify.

Could not verify from this session

This environment's outbound network access is restricted at the infrastructure level, so requests to theclarityshop.se and even the store's .myshopify.com address were blocked before reaching the site (confirmed as a proxy-level policy denial, not a problem with the store itself). That means the following couldn't be checked directly and should be verified manually:
  • Page speed / Core Web Vitals — check with PageSpeed Insights
  • Actual rendered <html lang>, canonical tags, and structured data (Product/FAQ schema) in the live theme
  • robots.txt contents and whether anything important is accidentally blocked
  • sitemap.xml — Shopify generates this automatically, but worth confirming it's submitted in Google Search Console
  • Whether Google Search Console and Google Analytics are connected and verified for this domain
  • Mobile rendering / usability on the live theme
